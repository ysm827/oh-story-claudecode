## browser-cdp 采集流程

### 通用流程

```
1. 启动 browser-cdp，打开目标榜单页面
2. 等待页面加载完成（检查列表元素出现）
3. 提取列表页数据（排名、书名、作者、字数、推荐/在读等）
4. 判断是否需要翻页：
   - 起点榜单通常单页展示完整（50-100条）
   - 番茄按题材逐页采集，每页 cap≈20
5. 逐条进入详情页获取补充数据：
   - 标签/标签区
   - 简介全文
   - 最新更新信息
6. 汇总写入 Markdown 文件
```

### 起点采集要点

- 榜单页直接包含：排名、书名、作者、题材、字数、推荐数、签约状态、免费/VIP
- 详情页需要提取：标签、最新更新、简介
- 三江榜按周分组，需识别周标题边界
- 简介可能包含换行和格式，原文保留

### 番茄采集要点

- 按题材逐个采集，每个题材一个 URL：`/rank/{channel}_{type}_{cat_id}`
- 题材列表可从榜单页面的分类导航动态提取（`a[href*="/rank/{channel}_{type}_"]`）
- 在读数是番茄核心指标（如"34.1万"）
- 单页展示上限约 20 本，需滚动加载更多
- **字体反爬**：页面使用自定义字体混淆，innerText 返回乱码。需用 `scripts/fanqie-rank-scraper.js`（通过详情页获取可读标题+作者，绕过字体反爬），或手动通过详情页 `/page/{book_id}` 获取可读文本
- 采集流程：先访问任一品类页 → 提取全部品类链接 → 逐品类切换采集

### 七猫采集要点

- 榜单页直接包含：排名、书名、作者、题材、分类标签、字数、热度、最新更新、简介摘要
- 男生榜/女生榜通过页面 tab 切换
- 大热榜有日榜/月榜切换
- 无明显反爬，但需要滚动加载更多条目
- 热度是七猫核心指标（如"156.5万"）

### 晋江采集要点

- 榜单页结构较老，排版密集，需要精确定位
- 收藏数、营养液、积分是核心指标
- 排行榜入口：`https://www.jjwxc.net/topten.php?orderstr={榜单ID}&t={频道ID}`
- 部分数据需登录态，browser-cdp 可复用已有登录

## 文件命名规范

```
{平台}{榜单名称}_{日期}.md
```

示例：
- `起点新人签约新书榜_20260425.md`
- `起点三江近四周_20260425.md`
- `番茄男频阅读榜_全题材_20260425.md`
- `七猫男频大热榜_20260425.md`
- `晋江收入金榜_20260425.md`

## 批量采集模式

用户说"全榜扫描"时：

| 平台 | 默认采集组合 |
|------|-------------|
| 起点 | 新人签约新书榜 + 月票榜前20 + 畅销榜前20 |
| 番茄 | 男频阅读榜全题材 + 女频阅读榜全题材 |
| 七猫 | 男频大热榜日榜 + 女频大热榜日榜 |
| 晋江 | 收入金榜 + 月榜 |
| 全平台 | 起点默认组合 + 番茄默认组合 + 七猫默认组合 |

每组采集完成后自动保存文件，再进入下一组。

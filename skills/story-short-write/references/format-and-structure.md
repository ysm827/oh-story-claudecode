# 短篇格式规范与小节结构

> 基于 8 篇平台真实小说统计（黑岩/点众/知乎）。写作前必读。

---

## 章节标记

推荐格式（按灵活度排序）：

| 格式 | 平台适用 | 示例 |
|------|----------|------|
| `###1.` | 黑岩、点众（主流） | `###1.` `###2.` `###3.` |
| `###第一章` | 部分平台 | `###第一章` `###第二章` |
| `1.`（纯数字） | 知乎 | `1.` `2.` `3.`（无 ### 前缀） |

**规则**：全文统一一种格式，不要混用。短篇推荐 `###1.` 或纯数字，简洁高效。

---

## 段落格式

### 核心规则：一句一段

平台短篇小说的标准排版是**一句一段，紧密排列**。

- 每段 1-3 句，不超 60 字
- 段与段之间**不加空行**（紧密排列）
- 无缩进（平台渲染器自行处理，不需要 `　　` 或空格）
- 一段只做一个动作：叙事/对话/心理/动作，不混合

### 段落节奏

连续的短段落（1-2 句）制造紧张感。偶尔用 3 句的段落作为呼吸点。但**绝不**出现超过 60 字的大段落。

---

## 对话格式

### 对话标记

基于 8 篇小说统计（5/8 用 `""`，1/8 用 `「」`）：

| 优先级 | 格式 | 适用平台 |
|--------|------|----------|
| 首选 | `"说话内容"` | 黑岩、点众、番茄（主流） |
| 次选 | `「说话内容」` | 部分古言、日式 |
| 平台指定 | `【说话内容】` | 仅知乎盐言故事 |

**默认用 `""`**，除非用户指定知乎平台则改为 `【】`。

### 对话规则

1. 对话**独立成行**，不嵌在叙述段落中
2. 用动作代替对话标签：不用「他说」「她道」，改为动作描写 + 对话
3. 两人对话连续出现时，省略标签，靠内容区分说话人

**正确示例**：
```
她把杯子放下。
"你走吧。"
他没有动。
"我说，你走吧。"
```

**错误示例**：
```
她把杯子放下，说道："你走吧。"他没有动，她又说："我说，你走吧。"
```

## 小节（beat）结构

### 基本规则

- 用数字编号（`1` `2` `3`）分割小节，每个小节是一个完整的叙事 beat
- 每小节 800-1500 字（爽文可压缩至 500-800 字/节，见 genre-writing-formulas.md）
- 8-15 个小节覆盖全文（按目标字数反推；8000÷1000≈8节，12000÷800≈15节）
- 每小节推进一个明确的情节点

### 小节内部结构

每个小节应该有：

1. **一个主事件** + **3-5 个子事件**（主事件推进核心情节，子事件丰富层次。子事件不足 3 个时用 writing-craft.md 第 7 节扩展规则）
2. **一个情绪变化**（读者的感受如何变化）
3. **一条读者新获知的信息**
4. **3-5 轮对话交锋**（揭示关系/升级冲突，参考 writing-craft.md 对话权力博弈。特定场景如独自发现/翻阅日记可标零，见 SKILL.md 设计任务第 4 步）
5. **每个子事件三层展开**：发生+感知+反应（参考 writing-craft.md 场景展开法）

小节偏短时的诊断方法见 writing-craft.md「小节密度诊断」和「场景展开法」。

### 小节之间的衔接

- 小节结尾留一个钩子（悬念/未解决的情绪/新问题）
- 下一节开头快速接续，不要重新铺垫
- 情绪跨节递进：每一节的情绪强度 ≥ 上一节。例外：峰值情绪（反转节）后允许维持 1 节不降，但不允许骤降

## 平台对话格式覆盖表

| 平台 | 章节标记 | 对话格式 | 特殊要求 |
|------|----------|----------|----------|
| 黑岩 | `###1.` | `""` | 无 |
| 点众 | `###1.` | `""` | 无 |
| 知乎盐言 | `1.` | `【】` | 导语需单独标注 |
| 番茄 | `###第一章` | `""` | 首段需有吸引力 |
| 红果 | `###1.` | `""` | 无 |

**通用原则**：用户未指定平台时，默认使用黑岩/点众格式（`###1.` + `""`）。

---

## 5 条绝对禁止

以下规则在写作全程执行，不因题材或风格而变：

1. **禁止大段落**：超过 60 字的段落必须拆分
2. **禁止段间空行**：段落紧密排列，不留空行
3. **禁止对话标签化**：不写「他说」「她道」「他笑了笑说」，用动作代替
4. **禁止缩进**：不使用 `　　`（全角空格）或半角空格缩进
5. **禁止 Markdown 渲染**：正文中不使用加粗 `**`、斜体 `*`、标题 `#` 等 Markdown 语法

写完每一段后，用这个清单自查：

- [ ] 每段 ≤ 60 字？
- [ ] 段间无空行？
- [ ] 对话独立成行且用正确引号？
- [ ] 无「他说/她道」？
- [ ] 无缩进？
- [ ] 每个小节有事件+情绪+信息？
